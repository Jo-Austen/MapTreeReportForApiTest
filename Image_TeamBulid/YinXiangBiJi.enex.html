泛型出现的背景

  • 指定容器中对象类型时定义：Collection<String> ct = new ArrayList<String>()
  • 当需要在容器中存储不同类型的对象时，之前的解决办法是将类型定义为Object，Object是所有类型的父类。此时，当需要对元素使用具体的方法时，需要对元素进行类型转换，不同数据类型元素转换成同一个类型时会出现问题。由于编译时，程序无法检测类型能否正常转换，只有在实际执行时才会报错，因此存在类型不安全的问题

https://segmentfault.com/a/1190000005337789#articleHeader0

public class Generic01 {
    public static void main(String[] args) {
        Collection ct = new ArrayList();
        ct.add("hello");
        ct.add("world");
        ct.add("java");
        ct.add(100);
        ct.add(1.01);

        Iterator it = ct.iterator();
        while (it.hasNext()){
            //转为String类型是上转下，需要使用强制类型转换
            // 编译时没有报错，但是运行时会报错：ClassCastException: class java.lang.Integer cannot be cast to class java.lang.String
            String o = (String)it.next();
            System.out.println(o.equals("world"));
        }
    }
}

泛型的好处和概述

  • 泛型，相当于将原来具体的类型参数化，只有在具体使用时才传入具体的类型
  • 有泛型类，泛型方法，泛型接口
  • 把运行时的问题提前到了编译期间，避免了强制类型转换
  • 泛型中的类型可以用T，E，K，V等参数表示
  • 类型可以是String，类，包装类

泛型类

// 可以传递多个参数,表明该类中会使用到的类型
public class Generic02<T,K> {
    private T id;
    private T age;
    private K score;

    public Generic02() {
    }

    public Generic02(T id, T age, K score) {
        this.id = id;
        this.age = age;
        this.score = score;
    }

    public T getId() {
        return id;
    }

    public void setId(T id) {
        this.id = id;
    }

    public T getAge() {
        return age;
    }

    public void setAge(T age) {
        this.age = age;
    }

    public K getScore() {
        return score;
    }

    public void setScore(K score) {
        this.score = score;
    }
}

public class Generic02Demo {
    public static void main(String[] args) {
        // 将类型的实参传递
        Generic02<Integer,Integer> gs = new Generic02<Integer, Integer>();
        gs.setAge(15);
        gs.setId(01);
        gs.setScore(95);
        System.out.println(gs.getId()+","+gs.getAge()+","+gs.getScore());

        Generic02<String,Integer> gt = new Generic02<String, Integer>();
        gt.setAge("18");
        // gt.setAge(18);会报错，因为限制了类型为String
        gt.setId("02");
        gt.setScore(100);
        System.out.println(gt.getId()+","+gt.getAge()+","+gt.getScore());
    }
}

泛型接口

public interface Generic04<T,K>{
    public void show(T t,K k);
}

public class Generic04Impl<T,K> implements Generic04<T,K> {
    @Override
    public void show(T t, K k) {
        System.out.println(t);
        System.out.println(k);
    }
}

public class Generic04Demo {
    public static void main(String[] args) {
        Generic04<String,Integer> g04 = new Generic04Impl<String, Integer>();
        g04.show("lily",05);
        g04.show("lucy",04);

        Generic04<String,String> g040 = new Generic04Impl<String, String>();
        g040.show("lily","047");
        g040.show("lucy","045");

        Generic04Impl<String,String> g041 = new Generic04Impl<String, String>();
        g041.show("lily","005");
    }
}

类型通配符

  • 类型参数<T>用于在声明泛型时使用，而类型通配符在使用泛型类或者泛型方法时使用
  • 由于集合可以存储任意类型的数据，因此其接口和实现类的参数都为泛型
  • <?>表示任意类型
  • <? extends 类型A>：表示接受类型A及其子类
  • <? super 类型A>：表示接受类型A及其父类
public class Generic05 {
    public static void main(String[] args) {
        // 可以创建任意类型的对象，创建的对象不能添加元素
        List<?> list01 = new ArrayList<>();
        List<?> list02 = new ArrayList<String>();
        List<?> list03 = new ArrayList<Number>();
        // list02.add("lily");
        list02.remove("lily");
        list02.size();
        // list02.set(0,"hello");

        // 可以创建本身及其子类类型对象，无法添加元素
        //List<? extends Number> list04 = new ArrayList<Object>();
        List<? extends Number> list05 = new ArrayList<Number>();
        List<? extends Number> list06 = new ArrayList<Integer>();
        //list05.add(15);
        list05.remove(15);
        //list05.set(0,15);
        list05.size();

        // 可以创建本身及其父类对象，可以添加元素
        List<? super Number> list07 = new ArrayList<Object>();
        List<? super Number> list08 = new ArrayList<Number>();
        //List<? super Number> list09 = new ArrayList<Integer>();
        list08.add(15);
        list08.remove(15);
        list08.size();
        list08.set(0,15);

    }
}

可变参数

  • 参数个数可变时使用可变参数表示，type... 变量
  • 这里的可变参数是一个列表
  • 包含可变参数且有多个参数时，可变参数放在最后，方便JVM确定可变参数的范围为余下的所有参数
public class Generic06 {
    public static void main(String[] args) {
        sum(1,2,3,4,5);
    }

    public static void sum(int a,int... b){
        int sum = 0;
        // 可变参数实质上是数组
        for (int i =0;i<b.length;i++){
            sum+=b[i];
        }
        System.out.println(sum);
    };
}

public class Generic07 {
    public static void main(String[] args) {
        //Arrays工具类中的方法：返回大小固定的列表，无法增删元素
        List list = Arrays.asList("hello","world","java",123);
       // list.add("ll");//UnsupportedOperationException
        //list.remove("hello");//UnsupportedOperationException
        list.set(0,"hi");
        System.out.println(list);

        List<String> list1 = Arrays.asList("hello","world","123");
       // list1.add("ll");//UnsupportedOperationException
        System.out.println(list1);

        //List接口中的方法，返回固定大小且内容不可变的列表
        List<String> list2 = List.of("hello","world","java","lily");
       // list2.add("hi");//UnsupportedOperationException
        //list2.remove("hi");//UnsupportedOperationException
        //list2.set(0,"hi");//UnsupportedOperationException
        System.out.println(list2);

        //Set接口中的方法，返回大小固定且内容不可变的集合
        Set<String> set = Set.of("hello","world","java");
        //set.add("hi");//UnsupportedOperationException
        //set.remove("hi");//UnsupportedOperationException
        System.out.println(set);
        
    }
}

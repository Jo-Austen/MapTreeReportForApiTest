Collection

常用方法

  1. Boolean add(element)：添加元素，输出结果永远为true
  2. Boolean remove(element): 移除元素，有则删除，并返回true，无则返回false
  3. void clear(): 清空集合中的元素，无返回值
  4. Boolean contains(element): 判断是否存在指定元素，存在返回true，不存在返回false
  5. Boolean isEmpty(): 判断是否为空，空为true，非空为false
  6. int size(): 返回长度

可用的迭代方法

  1. Iterator<String> it = ct.iterator(); 通过方法创建迭代器，通过迭代器的方法遍历并返回元素：hasNext()，next()
  2. 增强for循环遍历
  3. 普通for

集合的并发修改异常

  • http://www.manongjc.com/article/48905.html
  • ConcurrentModificationException()：并发修改异常
  • 产生原因：迭代器遍历过程中，通过集合对象修改了集合的长度（add、remove，修改单个元素的内容不会影响），使得迭代器获取到的预期修改次数和集合实际修改次数不一致
  • 解决方法：1. 不采用迭代器迭代，而采用普通for循环；2. 使用迭代器自带的add、remove方法修改

        ListIterator<String> lt = list.listIterator();
        while (lt.hasNext()){
            String s = lt.next();
            if (s.equals("world")){
                lt.add("beautiful"); //在world后面加beautiful
            }
        }


        System.out.println("________________");

        //for循环+get
        for(int i =0;i<list.size();i++){
            String s = list.get(i);
            if (s.equals("world")){
                list.add("beautiful"); //在集合的最后加beautiful
            }
        }

2. Set

  • 不包含重复元素，没有带索引的方法，只能通过迭代器遍历
  • 其实现类内部元素部分有序，部分无序
  • 继承自collection，含有collection的方法
  • 是接口，不能直接实例化，可以用其实现类HashSet和TreeSet实现其方法

2.1 HashSet

  • 底层数据结构是哈希表
  • 不保证存储和取出元素的顺序一致，和Set一样，无法使用索引，也不包含重复元素（根据哈希值，内容判断）
  • 默认长度是16，即0-15
  • 内容相同的new对象，内存地址不同。可以重写equals和hashcode方法，从而可以使得内容相同的对象无法重复

2.2 LinkedHashSet

  • 哈希表和链表实现的Set结构
  • 哈希表保证了元素的唯一性（无法存储重复元素）
  • 链表保证的元素存入和取出顺序的一致性（有序）
  • 没有索引方法

2.3 TreeSet

  • 没有带索引的方法
  • 不包含重复元素
  • 根据构造方法而按照一定规则排序，适用于需要自定义排序的情况
    • TreeSet()：无参构造根据元素Comparable接口的compareTo方法确定排序
    • TreeSet(Comparator comparator)：根据指定的比较器进行排序
